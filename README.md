# Enve Support

Public support, issue tracking, known issues, and release communication for the Enve app ecosystem.

## What this repo is for

This is the public support hub for Enve apps.

Use this repo to:

- report bugs
- request features
- ask support questions
- track known issues
- follow public fix progress
- see release-related updates

GitHub is the public source of truth. Discord is the faster day-to-day control surface used to manage and mirror that work.

## Covered apps

This support repo currently tracks issues for:

- Enve Book Player for iOS
- Enve Book Player for Android
- Enve Music Player for iOS
- Enve Video Player for iOS
- Enve Video Player for Android
- Enve RomM Player for iOS

Each public issue is tagged with an app label such as `app:book-ios` or `app:music-ios`.

## How to report something

Open an issue and include:

- the affected app
- app version
- device and OS version
- what happened
- what you expected
- steps to reproduce it
- screenshots or screen recordings if they help

If you use the Discord support flow, the Enve Librarian bot can create the GitHub issue for you automatically.

## Issue priority and community impact

Some issues are prioritized partly by how many people are affected and how severe the issue is.

When a report is created through Discord, users can:

- mark that they are affected
- rate severity from `1` to `10`

That lets the support flow surface problems that are both widespread and disruptive.

## Labels

Common labels used here:

- `app:*` identifies the affected app
- `report:*` identifies the report type
- `severity:*` records the initial reported severity
- `bug` means broken behavior
- `enhancement` means feature request or improvement
- `support` means troubleshooting or usage help
- `known-issue` means the issue is confirmed and being tracked publicly
- `app-store` means the report originated from App Store feedback

## Known issues and fixes

Confirmed issues may be marked with `known-issue`.

When a fix is shipped or a workaround is available, the issue will be updated and closed when appropriate. Release notes and public updates may also be posted here as issues, pinned items, or linked release announcements depending on the type of update.

## Public and private tracking

This repo is public so users can follow progress and participate in support.

Internal implementation work may still happen in private app repositories. When that happens, this repo remains the public-facing record for the issue, status, and release outcome.
